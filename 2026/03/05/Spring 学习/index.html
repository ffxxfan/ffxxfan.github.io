0x01 什么是 Spring

0x0101 Spring 定义

0x0102 为什么使用 Spring

  • IoC 1（控制反转）：管理对象的创建和依赖关系，开发者无需手动 new 对象。
  • AOP（面向切面编程）：把日志、事务等通用逻辑抽离成切面，自动作用到业务方法上，核心是代码复用和关注点分离。
  • 丰富的功能：涵盖数据访问、事务管理、MVC Web 框架、测试、安全等。
  • 强大的社区和生态系统：拥有庞大的用户群体、丰富的文档和第三方库支持。
  • 与主流技术无缝集成：如 Hibernate, MyBatis, Redis, Kafka, Elasticsearch 等。
  • Spring Boot：极大地简化了 Spring 应用的初始搭建和开发过程。

0x02 Spring 的核心架构

0x0201 核心基础 Spring Core Container

这是 Spring 最核心的模块，所有其他模块都依赖它，核心作用是 实现 IoC/DI，管理所有 Bean 的生命周期。你可以把它理解成 “Spring 的大脑”，负责创建对象、注入依赖、管理对象的存活。

子模块 功能
Beans 2 定义了 BeanFactory（IoC 容器的最基础接口），负责 Bean 的创建、注册、获取；
Core 提提供 IoC 3/ DI 4 的最基础功能（比如资源加载、类型转换）；
Context5 构建于 Core 和 Beans 模块之上，提供上下文信息（ApplicationContext）。
SpEL (Spring Expression Language) 一种运行时表达式语言，能通过表达式操作对象、取值、调用方法。

核心总结：Core+Beans 是 IoC 的 “底层引擎”，Context 是你实际用的 “上层容器”，SpEL 是 “操作容器内对象的工具”。

0x0202 通用增强 AOP Messaging

0x020201 AOP and Instrumentation（AOP 增强）

实现 AOP（面向切面编程），提供 “横切逻辑增强” 的能力；Instrumentation 是辅助功能（类加载增强、字节码操作），日常开发很少直接用。就相当于给你的业务方法 “加外挂”—— 比如给所有数据库操作自动加事务、给所有接口自动加日志，不用改业务代码。

  • 用@Transactional注解实现声明式事务（底层是 AOP）；
  • 自己写@Aspect切面实现日志、权限校验（之前给你写的 LogAspect 就是用这个模块）；
  • Spring 的缓存注解@Cacheable、异步注解@Async，底层都是 AOP 模块支撑。

0x020202 Messaging（消息模块）

支持消息传递相关的标准和协议，重点是 STOMP（简单文本定向消息协议）和 WebSocket，解决 “实时双向通信” 问题，比如做聊天功能、实时推送（如订单状态更新）。

0x0203 业务场景 Data Web

0x020301 Data Access/Integration（数据访问 / 集成）

子功能 功能
JDBC6 简化原生 JDBC 操作（比如封装JdbcTemplate，不用手动写 Connection/Statement）
ORM7 集成主流 ORM 框架（JPA、Hibernate、MyBatis）
OXM 对象 - XML 映射（比如把 Java 对象转 XML，或反过来）
JMS8 集成 JMS（Java 消息服务），对接 ActiveMQ、RabbitMQ 等消息队列
Transactions 事务管理（声明式 / 编程式）

核心总结：只要你做 “数据库操作” 或 “消息队列交互”，就离不开这个模块；Spring Boot 中引spring-boot-starter-data-jpa/spring-boot-starter-mybatis，会自动把这个模块的核心功能引入。

0x020302 Web（Web 模块）

专门负责 Web 应用开发，核心是 Spring MVC，也支持 WebSocket（和 Messaging 模块配合）。

  • 实现 MVC 架构（Controller 接收请求→Service 处理业务→View 返回响应）；
  • 处理 HTTP 请求、参数绑定、视图解析、拦截器等；
  • 支持 WebSocket（实时通信）。

0x0204 辅助工具 Test

简化 Spring 组件的单元测试 / 集成测试，整合 JUnit/TestNG，提供模拟（Mock）功能。

  • 用@SpringBootTest注解启动 Spring 上下文做集成测试，能自动注入@Service/@Controller；
  • 用MockMvc模拟 HTTP 请求，测试接口是否正常（不用启动 Tomcat）；
  • 用@MockBean模拟依赖的 Bean（比如测试 UserService 时，模拟 OrderService，避免依赖真实数据库）。

0x03 实际使用逻辑

不用刻意背每个模块的细节，实际开发中，`Spring Boot 已经帮你做了 “模块封装”—— 通过 Starter 依赖，自动引入对应模块：

开发场景 引入的 Starter 底层包含的 Spring 模块
基础 Spring Boot 项目 spring-boot-starter Core Container（核心容器）
Web 接口开发 spring-boot-starter-web Core Container + Web
数据库操作（MyBatis） spring-boot-starter-mybatis Core Container + Data Access + AOP
声明式事务 spring-boot-starter-jdbc Core Container + Data Access + AOP
测试 spring-boot-starter-test Core Container + Test
Spring AI + RAG spring-ai-rag Core Container + Web（可选） + AOP（可选）

0x0402 分层架构模式

控制层（Controller）

  • “前端的对接窗口”
  • 不写业务逻辑，只做 “请求接收 - 参数校验 - 调用 Service - 返回响应”；
  • 核心注解：@RestController、@GetMapping/@PostMapping；

服务层（Service）

  • 核心注解：@Service、@Transactional；
  • 核心原则：所有业务逻辑都写在这里，是项目的 “核心大脑”，实现核心业务逻辑（比如下单、支付、用户注册），调用数据访问层操作数据库；

数据访问层（DAO/Mapper）

  • 核心注解：MyBatis 用@Mapper，JPA 用@Repository，JdbcTemplate；
  • 核心原则：只写 SQL / 数据库操作，只负责和数据库交互（增删改查），不包含任何业务逻辑；

实体层（Entity/Model）

  • 封装数据，作为各层之间的数据传输载体；

  • 分三层

    类型 作用 使用场景
    Entity（实体） 对应数据库表结构 DAO 层和数据库交互时用
    DTO（数据传输对象） 对应前端请求 / 响应的参数 Controller 层和前端交互时用
    VO（视图对象） 封装前端展示的数据 复杂页面展示（比如列表页）

  • 核心注解：Lombok 的@Data（简化代码）；

  • 核心原则：不同场景用不同的实体类，避免一个类走天下（比如前端不需要的字段不传给前端）；

工具层（Util）

  • 常用工具：日期处理、字符串处理、加密、全局异常处理、常量类（比如Constants）；
  • 提供全局异常处理、常量定义等；
  • 不依赖其他业务层，通用、无状态。

0x05 常用注解速查表

0x0501 常用注解速查表

注解 用途 所在模块
@Component 通用组件注解 spring-context
@Service 服务层组件 spring-context
@Repository 数据访问层 (DAO) 组件，能捕获持久化异常 spring-context
@Controller 表现层 (Web) 组件 spring-web
@RestController @Controller + @ResponseBody，用于 RESTful API spring-web
@Configuration 声明配置类 spring-context
@Bean 在配置类中声明一个 Bean spring-context
@ComponentScan 启用组件扫描 spring-context
@Autowired 自动装配依赖（构造函数、Setter、字段） spring-beans
@Qualifier 当有多个相同类型的 Bean 时，指定名称进行注入 spring-beans
@Value 注入属性值（字面量、SpEL 表达式） spring-beans
@Scope 定义 Bean 的作用域 (singleton, prototype 等) spring-beans
@Primary 当有多个候选 Bean 时，优先选择此 Bean spring-beans
@PostConstruct 定义初始化方法 javax.annotation
@PreDestroy 定义销毁前方法 javax.annotation

0x0502 常用注解速查表（按分层架构模式）

分层名称 常用注解 注解具体用途
控制层（Controller） @RestController 标记为 REST 接口控制器，返回 JSON 数据（替代@Controller + @ResponseBody）
@Controller 标记为传统 MVC 控制器，返回视图（如 HTML 页面）
@RequestMapping 映射请求路径（可指定请求方法），用于类 / 方法上（如@RequestMapping("/user")）
@GetMapping("/xxx") 专门映射 GET 请求（查询类接口），简化@RequestMapping(method=GET)
@PostMapping("/xxx") 专门映射 POST 请求（新增 / 修改类接口）
@RequestParam 获取 URL 中的键值对参数（如/user?name=张三）
@PathVariable 获取 URL 路径中的参数（如/user/{id}）
@RequestBody 获取请求体中的 JSON 数据，绑定到 DTO 对象（POST 请求专用）
服务层（Service） @Service 标记类为 Spring 管理的业务层 Bean，让 Spring 创建并管理实例
@Transactional 给方法 / 类添加声明式事务，保证数据库操作原子性（出错自动回滚）
@Autowired 自动注入依赖的 Bean（如注入 DAO），推荐用构造注入替代字段注入
数据访问层（DAO/Mapper） @Mapper MyBatis 注解，标记接口为数据访问层，自动生成实现类（无需写 XML）
@Repository JPA 注解，标记类为数据访问层 Bean，Spring 自动管理
@Select("SQL") MyBatis 注解，执行查询 SQL（替代 XML 中的 select 标签）
@Insert("SQL") MyBatis 注解，执行插入 SQL
@Update("SQL") MyBatis 注解，执行更新 SQL
@Delete("SQL") MyBatis 注解，执行删除 SQL
实体层（Entity/DTO/VO） @Data Lombok 注解，自动生成 get/set/toString/equals 等方法，简化代码
@NoArgsConstructor Lombok 注解，生成无参构造方法（Spring/MyBatis 反射需要）
@AllArgsConstructor Lombok 注解，生成全参构造方法（方便创建对象）
@Entity JPA 注解，标记类为数据库表对应的实体类
工具层（Util） @Component 标记通用工具类为 Spring Bean（如全局异常处理器、自定义工具类）
@RestControllerAdvice 全局异常处理器，捕获 Controller 层所有异常并统一返回格式
@ExceptionHandler 配合@RestControllerAdvice，指定捕获的异常类型（如Exception.class）
