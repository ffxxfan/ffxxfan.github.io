计算机相关的专有名词解释

0x01 计算机网络相关

DNS: Domain Name System，是域名系统。DNS 就是把域名解析为 IP 地址，提供我们上网，我们能够上网最终是找到 IP 地址。比如，https://ffxxfan.github.io/(ffxxfan.github.io) 是域名，那么它的 IP 地址假设是 202.117.80.6，我们用浏览器访问该的网页，先到 DNS 服务器找 https://ffxxfan.github.io/ 对应的 IP 地址，然后通过 IP 地址进行访问。

DNS 污染：DNS cache pollution。又称 域名服务器缓存投毒（DNS cache poisoning），是指一些刻意制造或无意中制造出来的 域名服务器（DNS） 数据包，把域名指往不正确的 IP 地址。一般来说，在互联网上都有可信赖的网域服务器，但为减低网络上的流量压力，一般的域名服务器都会把从上游的域名服务器获得的解析记录暂存起来，待下次有其他机器要求解析域名时，可以立即提供服务。一旦有关网域的局域域名服务器的缓存受到污染，就会把网域内的计算机导引往错误的服务器或服务器的网址。

0x02 计算机视觉

Prompt：是用户提供给指向 模型（可以是 GPT 等语言模型 AI，也可以是 SAM 等分割模型）提供输入以引导其生成特定输出的文本或指令，用于 引导模型生成特定的输出。它可以是一个问题、一段指令、一个关键词，甚至是一段不完整的文本或者图像上的点。模型根据 prompt 理解用户的意图，并生成相应的回答或内容。

也就是说，模型会根据用户给的 prompt 生成用户想要的东西。

prompt 用于提示，不局限于形式，如 图像、点（如图像上的某个像素）、文本等，只要是能够给 模型 带来 「启发式思考」的东西，都可以称作 Prompt。

生成模型：生成模型 是一类能够学习数据分布并生成新数据的机器学习模型。它们可以模仿训练数据的统计特征，生成类似但全新的样本，如图像、文本、音频等。与 判别模型（Discriminative Models，如分类器）不同，生成模型的目标是建模数据的生成过程，而非仅仅区分不同类别。目前主流的方法有：GAN、Diffusion Models、VAE、flow matching 等。

图灵测试（The Turing Test）：起源于 计算机科学 和 密码学 的先驱 艾伦·麦席森·图灵 发表于 1950 年的一篇论文《计算机器与智能》。该测试的流程是，一名测试者写下自己的问题，随后将问题以纯文本的形式（如计算机屏幕和键盘）发送给另一个房间中的一个人与一台机器。测试者根据他们的回答来判断哪一个是真人，哪一个是机器。所有参与测试的人或 机器 都会被分开。这个测试旨在探究机器能否模拟出与人类相似或无法区分的智能。

赫布理论（Hebbian theory）：是一个 神经科学 理论，解释了在 学习 的过程中 脑 中的 神经元 所发生的变化。赫布理论描述了 突触可塑性 的基本原理，即突触前神经元向突触后神经元的持续重复的刺激，可以导致突触传递效能的增加。这一假说现已被神经生物学实验所证实。

表征学习（Representation Learning）：表征学习是机器学习的一个分支，其核心目标是自动从原始数据中提取有用的特征（即“表征”），使后续任务（如分类、回归、聚类等）更高效、更准确。表征学习关注如何自动找出表示数据的合适方式，以便更好地将输入变换为正确的输出，深度学习是表征方法的一种。

HSI（Human-Scene Interaction）：人类-场景交互，在计算机视觉、机器人控制、计算机图形学等领域中，HSI 指的是 人类与三维场景中的物体、地形或环境之间的物理交互行为建模与合成。HSI 涉及的交互行为很多，比如：

  • 走路跟随路径（Follow）
  • 坐在椅子上（Sit）
  • 攀爬障碍物（Climb）
  • 搬运物体（Carry）
  • 跳跃、抓取、摆放、推拉等日常交互

这些动作不仅要看起来自然，还要满足物理可行性（如平衡、重力、碰撞检测等），这就需要结合物理引擎和智能控制算法进行建模。

0x03 计算机基础

语法糖：语法糖 是指编程语言中为了方便开发者使用而提供的简洁、易读的语法形式，其功能可通过更基础的语法实现，旨在提升代码的可读性和编写效率。比如 Python 语法中 装饰器 使用 @decorator 简化手动包裹操作。

静态方法：属于类本身，调用时无需创建实例，直接通过类名访问，不能访问实例成员。

动态方法（实例方法）：属于类的实例，必须通过对象调用，可以访问实例的属性和方法。

MSVC：MSVC 是 Microsoft Visual C++ 的缩写，是微软开发的一套 C/C++ 编译器工具集，属于 Visual Studio 开发环境的一部分。

Windows SDK：Windows SDK（Windows 软件开发工具包） 是 系统功能接口包，核心功能是 “提供系统调用能力”：

  • 包含大量头文件（.h）、库文件（.lib），定义了 Windows 系统的各种功能接口（比如创建窗口、操作文件、调用系统 API 等）。
  • 提供了一些开发辅助工具（如资源编译器、调试工具）和系统相关的文档、示例。
  • 是 “让程序能调用 Windows 系统功能” 的基础，比如你想让程序显示一个窗口，就需要用 SDK 里定义的窗口相关接口。

0x04 数学相关

最小二乘法（Least Squares Method）：最小二乘法的目标是找到一组模型参数，使得 预测值与实际观测值之间的残差平方和（即误差平方和）最小。

数学表达式为： $$\min \sum_{i=1}^n (y_i - \hat{y}_i)^2$$

其中：

  • yi 是第 i 个实际观测值，
  • ŷi 是模型对第 i 个观测的预测值。

伯努利分布（Bernoulli Distribution）：是概率论中最基础的离散概率分布之一，描述了一个只有两种可能结果的单次随机试验（称为 伯努利试验）。

其特点如下：

  • 两种结果：试验结果只有两种，通常编码为：
    • 成功（1）：概率为 p  (0 ≤ p ≤ 1)。
    • 失败（0）：概率为 1 − p。
  • 典型例子：抛硬币（正面/反面）、二分类问题（是/否）、事件发生与否等。

0x0501 分类

Iris（鸢尾花）：鸢尾花数据集包含了三种鸢尾花（Setosa、Versicolor、Virginica）每种花的 4 个特征：花萼长度、花萼宽度、花瓣长度和花瓣宽度，仅 150 个样本。

MNIST：是一个通常用于 训练 各种 数字图像处理 系统的大型 数据库 [2] [3]。该数据库通过对来自 NIST 原始数据库的样本进行修改创建，涵盖手写数字的图像，共包含 60,000 张训练图像和 10,000 张测试图像，尺寸为 28×28 像素。

0x06 游戏引擎相关

时间超级分辨率：时间超级分辨率（Time Super-Resolution，TSR）是一种用于提升视频时间分辨率的技术。它不仅能够增加视频的帧率，还能够恢复超出原始视频帧率奈奎斯特极限的高频信息，从而有效解决运动模糊和运动混叠问题。传统的提高视频帧率的方法，如时间插值技术，通常只是简单地在现有帧之间插入额外的帧，无法恢复由于低采样率带来的高频信息。而时间超级分辨率算法则采用了不同的方法，例如通过观察视频帧中的小区域（空间时间补丁，ST-patches）在时间和空间上的冗余性和递归性，利用高分辨率的空间帧来指导提高时间切片的分辨率。此外，时间超级分辨率算法还可以采用零样本学习，直接利用输入视频本身作为训练数据，无需任何外部样本。通过生成内部训练集，让卷积神经网络（CNN）学习到与输入视频紧密相关的特定统计特性，从而更好地适应视频内容，提高模型对新数据的泛化能力。

Nanite 虚拟几何体：是虚幻引擎 5 的虚拟化几何体系统，它采用全新的内部网格体格式和渲染技术来渲染像素级别的细节以及海量对象。Nanite 可以智能地仅处理能被玩家感受到的细节，采用高度压缩的数据格式，并且支持具有自动细节级别的细粒度流送。使用 Nanite 后，几何体形状的复杂度大幅提高，三角形和对象的实时渲染数量达到了前所未有的高度，帧预算不再受多边形数量、绘制调用和网格体内存使用情况的限制，还可以直接导入电影级品质的美术资源，无需手动设置单个网格体的细节级别（LOD）。

虚拟阴影贴图：是一种全新的阴影映射方法，可以提供稳定的高分辨率阴影，通过与虚幻引擎 5 的 Nanite 虚拟几何体、Lumen 全局光照和反射以及世界分区功能结合使用，能够处理电影级品质的资产，为大型开放世界提供动态光照。虚拟阴影贴图的虚拟分辨率为 16k x 16k 像素，为了在合理内存开销下保证高性能，会将阴影贴图分成 128x128 个区块，系统会根据需要动态地分配并渲染页。此外，结合阴影贴图光线追踪（SMRT）算法，还能生成更为真实的柔和阴影和接触硬阴影。

蓝图：Blueprint（蓝图）是Unreal Engine 5中的一种可视化脚本系统，允许开发者通过拖拽节点图的方式来实现游戏逻辑，而无需编写代码。它非常适合设计师和艺术家快速实现和迭代游戏功能，同时也可与C++代码无缝集成。

  • 使用节点（Nodes）和线（Wires）连接，构建游戏行为和事件流程。
  • 支持创建角色移动、触发动画和音效、交互逻辑、用户界面（UMG）、关卡事件等。
  • 提供调试工具，方便查错和优化脚本。
  • 同时支持设计时和运行时使用。

派生数据：派生数据（Derived Data）是在 Unreal Engine 中由原始资产通过加工、编译等过程生成的中间或最终数据，用于加速资源加载和渲染。

  • UE资产（例如材质、纹理、着色器等）本身只是编辑时的源数据，但为了在游戏或编辑器中高效使用，这些资产需要被编译或转换成特定平台和硬件的格式，这些生成的结果就是派生数据。
  • 派生数据的存在能避免每次使用资产都重复进行耗时的编译或转换过程，从而提升启动和加载速度。

关卡编辑器：关卡编辑器（Level Editor）是 Unreal Engine 5 的核心编辑器窗口，用于构建和编辑游戏关卡（Levels），即游戏中的3D环境和场景。

  • 关卡包含所有玩家可见和交互的元素，比如地形、建筑、道具、角色、灯光和声音，这些元素统称为Actor。
  • 通过关卡编辑器，你可以放置、移动、旋转和调整这些Actor，设计游戏的玩法空间和视觉效果。
  • 关卡保存为.umap文件，是一个包含所有场景元素的容器。
  • 你可以用关卡编辑器实现环境搭建、布置交互对象、设置蓝图脚本、使用Niagara特效等。
  • 这就是你制作游戏世界或体验的主要工作区域，UE启动项目时默认打开此编辑器。

映射上下文：可以根据玩家的当前状态更改特定输入的行为。例如，如果你有一个可以行走、冲刺、俯卧的玩家角色。对于其中每种角色移动类型，你可以交换映射上下文，让CTRL键执行不同的操作。行走时按下CTRL键，角色应该蹲伏。冲刺时按下CTRL键，角色应该滑行。俯卧时按下CTRL键，角色应该重新站起来。

输入动作：输入动作（Input Actions） 是增强输入系统和项目代码之间的通信链接。输入动作在概念上相当于 操作（Action） 和 轴（Axis） 映射名称，但它们是数据资产。输入动作可以是多种不同的类型，这些类型将确定行为。你可以创建简单的布尔动作或更复杂的3D轴。

例如，“拾取道具”动作可能仅需要一个开/关状态，用于指示用户是否想要角色捡起某个东西，而“行走”动作可能需要2D轴，来描述用户想要角色行走的方向和速度。

触发状态：触发状态（Trigger State） 表示动作的当前状态，例如 已开始（Started） 、 进行中（Ongoing） 、 已触发（Triggered） 、 已完成（Completed） 和 已取消（Canceled）

  • 已触发（Triggered）： 动作已触发。这意味着它完成了所有触发器要求的求值。例如，“按下并松开”触发器会在用户松开按键时发送。
  • 已开始（Started）： 发生了开始触发器求值的某个事件。例如，“双击”触发器的第一次按键将调用一次“已开始”状态。
  • 进行中（Ongoing）： 触发器仍在进行处理。例如，当用户按下按钮时，在达到指定持续时间之前，“按住”动作处于进行中状态。根据触发器，此事件将在收到输入值之后在对动作求值时，每次更新触发一次。
  • 已完成（Completed）： 触发器求值过程已完成。
  • 已取消（Canceled）： 触发已取消。例如，在“按住”动作还没触发之前，用户就松开了按钮。

输入修饰器：输入修饰器（Input Modifiers） 是一种预处理器，能够修改UE接收的原始输入值，然后再将其发送给输入触发器（Input Trigger）。

输入修饰器很适合用于应用灵敏度设置，在多个帧上平滑输入，或基于玩家状态更改输入的行为。由于你在创建自己的修饰器时可以访问 UPlayerInput 类，你可以访问所属玩家控制器，并获取所需的任意游戏状态。

注入输入：增强输入（UE5的插件）还为玩家带来了“注入输入”的概念。这样你可以调用蓝图、C++中的函数，或使用控制台命令模拟玩家的输入。
