0x01 什么是 Python

Python 是一种解释型、面向对象、动态数据类型的高级程序设计语言。Python3 和 Python2 具有较大的差异，现在主要使用的是 Python3，比较稳定的版本是 3.9 版本（2025-05-07 更新），算是兼容比较多的版本。

0x02 环境搭建

我们可以使用 PyCharm 作为我们的编译器，并使用 Anaconda 作为 Python 包管理器。

由于 Python 有非常多的包（Packages，PyPI），我们需要根据不同的项目安装不同的包。但是项目 A 和项目 B 之间可能会存在包冲突，及 Package 1 和 Package 2 本身不能兼容或两者版本不能兼容，或者 Package 3 与当前 Python 版本不兼容，所以我们最好使用 Anaconda 来为每个项目管理 Python 版本和相关的包。

而 Anaconda 为每个项目隔离出来的 Python 版本和相关的包，我们称它们为一个 环境 ，所以我们应该为每一个项目先创建一个环境，再在该环境配置相关的 Python 版本和包。

具体来说（以下只是一种方式，按照自己的习惯来就好，我比较喜欢用 anaconda 创建环境只是因为我用 pycharm 直接创建会报错，不想找原因……能用就好 √），

  • 安装 Pytharm、Anaconda，并配置好相关环境变量；

  • 对于某一个项目，为它创建对应 环境 ，并指定 Python 版本；

  • 在 Pycharm 打开已有项目或空项目，在 设置 -> 项目 -> Python 解释器 -> 添加解释器 中添加上一步创建的环境的 python.exe ，具体位置你可以在  anaconda 根目录 -> env -> your_environment_name -> bin or 当前目录  中找到。

  • 通过 pip 安装对应的包，如果涉及 Pytorch 建议使用 官方命令进行安装。如果是已有项目，检查是否存在 requirement.txt 文件。

0x03 代码规范 更优雅的代码

你也可以不遵守用 abcd 来命名，但是本来可以很优雅的。

Python 的命名 PEP 8（Python Enhancement Proposal 8） 规范，以下是核心规则和常见场景的命名约定。

0x0301 代码布局

  • 行长度：每行不超过 79 字符（代码）或 72 字符（文档字符串/注释），即一行代码的首尾可以在一屏看见，换行使用反斜杠 \ 或括号自然延续；

  • 空行：函数/类之间：2 行；方法/逻辑块之间：1 行；

0x0302 命名规范

  1. 命名风格：

    • 小写字母 + 下划线（snake_case）：变量、函数、方法、模块、包

    • 首字母大写（PascalCase/CamelCase）：类名、异常

    • 全大写 + 下划线：常量

  2. 避免的命名：

    • 单字符（除非是临时变量或数学符号）：i, x（可接受在循环中）
    • 保留 关键字 冲突：如 list，str（可加下划线后缀 list_），
    • 模糊缩写：tmp（可接受），但避免 fn（应为 function）

  3. 具体场景

    类型 规范 示例
    变量/函数 snake_case user_count
    类 PascalCase DatabaseEngine
    常量 UPPER_CASE MAX_RETRIES = 3
    私有成员 _single_underline _internal_data
    魔法方法 __double_underline__ __init__

  4. 其他建议

    • 一致性优先：如果项目已有规范，优先遵循项目约定。

    • 描述性命名：

      • 推荐：file_path（明确）
      • 避免：fp（模糊）

    • 避免误导：

      • 布尔变量用 is_ 或 has_ 前缀：is_valid
      • 集合类型用复数：users（而非 user_list，除非强调类型）

0x04 基本数据类型

Python 是一种动态类型语言，声明变量不用指明类型，Python 会根据赋给变量的值自动进行推断。

Python3 中常见的六个标准数据类型：

  • 不可变类型（Immutable）： Number（数字）、String（字符串）、Tuple（元组）；

  • 可变类型（Mutable）： List（列表）、Dictionary（字典）、Set（集合）。

    不可变类型（Immutable）：创建后，值本身（内存中的内容）无法改变，「修改」操作实际是创建新对象，可作为字典的键或集合的元素；

    可变类型（Mutable）：无需创建新对象即可增删改内容，不能直接作为字典的键或集合的元素（需转为不可变类型，如 tuple 或 frozenset）。

    其中，Number（数字）包括 int、float、bool、complex（复数）四种。

0x0401 基本数据类型概览表

哈希性

  • 可哈希（Hashable）对象才能作为字典的键或集合的元素（需实现 __hash__ 方法）；
  • 是否可哈希和可变性并不相关，尽管实际上 range 对象的内容在创建后是不可变的，但 Python 仍将其归类为不可哈希类型；

常见误区

  • 元组不可变，但若包含可变元素（如列表），其内容仍可修改；
  • 变量名是引用，重新赋值（如 x = 10 → x = 20）是改变引用，而非修改对象；

0x0402 基本数据类型

0x0402A Number bool

Python3 支持 int、float、bool、complex（复数），绝大部分都和其他编程语言一样，在此不做过多赘述，部分值得注意的是：

  • 数值的除法包含两个运算符：/ 返回一个浮点数，// 返回一个整数。// 得到的并不一定是整数类型的数，它与分母分子的数据类型有关系。

  • 在 Python 中，number（数字类型，如 int、float）是 不可变（immutable） 的。这意味着一旦创建了一个数字对象，它的值就不能被修改。

    a = 10 a = 20 # 这里不是修改 10，而是让 a 指向一个新的数字对象 20

  • 所有非零的数字和非空的字符串、列表、元组等数据类型都被视为 True，只有 0、空字符串、空列表、空元组 等被视为 False。因此，在进行布尔类型转换时，需要注意数据类型的真假性。

0x0402C List []

list 是 Python 的“万能动态数组”，灵活但需注意 复制、遍历修改和性能 问题！和 字符串 一样，列表同样可以被索引和截取，列表被截取后返回一个包含所需元素的新列表。列表截取的语法格式 变量[头下标: 尾下标: 步长]，索引值以 0 为开始值，-1 为从末尾的开始位置。

  • List 可以使用 + 操作符进行拼接；

  • List 中的元素是可以改变的，是不可哈希的；

  • List 相比字符串内置了有很多方法，例如 append()、pop() 等等；

  • append() 是 O(1) 操作（高效）；insert(0, x) 是 O(n) 操作（头部插入慢，需移动所有元素）；需要频繁头部操作时，考虑用 collections.deque。

非常值得注意的是：

  1. 列表复制问题：

    • 直接赋值 = 是 浅拷贝（共用同一内存），修改会影响原列表！

    • 正确做法：用 copy() 或 list() 复制新列表。

  2.  循环中修改列表：

    • 边遍历边修改列表可能导致意外结果或报错。

    • 正确做法：遍历副本或新建列表。

0x0402D tuple ()

tuple 是 Python 中的一种不可变序列，可以理解为一个 “一旦创建就不能修改的固定清单”。

  • 不可变（Immutable）：创建后 不能增、删、改 元素。
  • 有序：元素按添加顺序排列，可以通过索引（变量[头下标: 尾下标: 步长]）访问。
  • 可存任意类型：和列表一样，支持数字、字符串、列表等。
  • 允许重复：同一个值可以出现多次。

非常值得注意的是：

  1.  单元素元组的陷阱：如果元组只有一个元素，必须加逗号，否则会被当成普通变量！

  2. 元组内的可变对象：元组本身不可变，但如果元素是可变对象（如列表），内部内容仍可修改！

  3. 性能比列表略高：

    • 由于不可变性，元组的 存储和访问速度比列表快，适合存储常量数据。

    • 在需要哈希（如字典的键）时，元组比列表更合适（列表不可哈希）。

0x0402E set

set 是 Python 中的一种 无序、不重复 的数据容器，可以理解为一个 “自动去重的数学集合”。

  • 无序：元素没有固定顺序（不能通过索引访问）。
  • 唯一性：自动去除重复值（例如 {1, 2, 2} 会变成 {1, 2}）。
  • 可变（Mutable）：可以增删元素，但元素本身必须 不可变（如数字、字符串、元组）。
  • 高效查询：用哈希表实现，检查元素是否存在比列表快得多（O(1) 时间复杂度）。

s1 & s2（交集）

快速去重 list(set([1, 2, 2])) 列表 → 集合 → 列表

值得注意的是：

  1. 空集合必须用 set()，不能用 `{}

  2. 集合只能存不可变对象

    • 合法：数字、字符串、元组（如果元组内不含可变对象）。

    • 非法：列表、字典、其他集合（因为它们可变）。

  3. 集合是无序的，不要依赖顺序，每次打印的顺序可能不同（Python 3.7+ 后字典有序，但集合仍无序）。

0x0402F Dictionary

dict 是 Python 中的 键值对集合，可以理解为一个 “现实生活中的字典”（通过“单词”查“释义”），dict 是 Python 的“万能查找表”，用键快速找值，适合存结构化数据或高频查询！记住键必须不可变，访问不存在的键用 .get() 防报错。

  • 键值对结构：每个元素由 key: value 组成（例如 {"name": "Alice", "age": 25}）。
  • 键必须不可变：key 可以是 str、int、tuple 等不可变类型，但不能是 list、dict 等可变类型。
  • 无序（Python 3.7+ 后按插入顺序保存，但不要依赖顺序）。
  • 可变：可以随时增删改键值对。
  • 查询极快：通过 key 找 value 的时间复杂度是 O(1)（哈希表实现）。

值得注意的是：

  1. 访问不存在的键会报错，应该用 .get() 方法提供默认值。 ：

  2. 字典的键必须不可变：

    • 合法键：str、int、float、tuple（如果元组内不含可变元素）。

    • 非法键：list、dict、set。

  3. 字典是无序的（即使 Python 3.7+ 保留插入顺序），

    • 如果需要有序字典，用 collections.OrderedDict。

    • 不要依赖遍历顺序（除非明确使用排序）。

  4. 避免在循环中修改字典，边遍历边增删字典可能导致意外结果，应该遍历副本或先收集要修改的键；

0x0402G bytes

Python 中的 bytes（字节）简单解释，bytes 是 Python 中表示 二进制数据 的不可变序列，可以理解为一串 “0 和 1 的原始数据”（比如文件、网络传输的原始格式）。

  • 不可变（Immutable）：创建后不能修改（类似 tuple）。
  • 存储二进制数据：如图片、音频、加密数据等。
  • 与 str 的区别：
    • str 是 文本（如 "hello"），用 Unicode 编码。
    • bytes 是 二进制（如 b"hello"），用 ASCII/UTF-8 等编码的原始字节。

bytes vs bytearray

特性 bytes bytearray
可变性 ❌ 不可变 ✅ 可变
语法 b"ABC" 或 bytes() bytearray(b"ABC")
用途 存储固定二进制数据 需要修改的二进制数据

什么时候用 bytes？

  1. 处理文件/网络数据：如图片、音频、PDF 等二进制文件。
  2. 加密/哈希运算：如 hashlib.md5(b"data") 需要输入 bytes。
  3. 底层协议通信：如 TCP/UDP 传输原始数据。

  1. 可变参数：

    • *args：接收任意数量的位置参数（打包为元组）。

    • **kwargs：接收任意数量的关键字参数（打包为字典）。

  2. 可变类型（Mutable）：

    • 常见类型：list, dict, set, bytearray。

    • 传递特点：

      • 函数内对参数的修改（如增删元素）会 直接影响原对象。
      • 若重新赋值（=`），则会让参数指向新对象，原对象不受影响。

  3. 不可变类型（Immutable）

    • 常见类型：int, float, str, tuple, bool, frozenset。

    • 传递特点：

      • 函数内修改参数的值时，实际上是创建了一个 新对象，原对象不受影响。
      • 函数外的变量仍指向原始对象。

  4. 可变类型（Mutable）与不可变类型（Immutable）总结

    行为 不可变类型（如 int, str） 可变类型（如 list, dict）
    函数内直接修改 创建新对象，原对象不变 直接修改原对象
    函数内重新赋值（=） 参数指向新对象，原对象不变 参数指向新对象，原对象不变
    内存地址变化 修改后地址改变 修改元素时地址不变，重新赋值时改变

  5. 注意事项

    • 避免意外修改可变对象：

      若需保护原数据，可传入拷贝（如 lst.copy() 或 copy.deepcopy()）。

0x0601 迭代器和生成器

在 Python 中，迭代器（Iterator） 和 生成器（Generator） 都是用于实现迭代协议的工具，允许按需生成或遍历数据，但它们有不同的实现方式和用途，两者都是 Python 迭代协议的核心组件，生成器因简洁性和高效性更常用。

  • 迭代器：自定义复杂遍历逻辑（如树结构遍历）。
  • 生成器：处理数据流（如读取大文件）、无限序列（如斐波那契数列）。

      • 内置函数 iter() 和 next() 会调用对象的 __iter__() 和 __next__() 方法。

    • 定义：生成器是一种 特殊的迭代器，通过函数中的 yield 关键字实现，可以暂停和恢复执行。

      • 惰性计算：按需生成值，节省内存。
      • 生成器函数调用时返回一个生成器对象（自动实现迭代器协议）。

0x0604 类和方法

具体的请学习面向对象课程，此处只做简单的介绍；

  • 类(Class): 用来描述具有相同的属性和方法的对象的集合。它定义了该集合中每个对象所共有的属性和方法。对象是类的实例。

  • 方法：类中定义的函数。

  • 类变量：类变量在整个实例化的对象中是公用的。类变量定义在类中且在函数体之外。类变量通常不作为实例变量使用。

  • 数据成员：类变量或者实例变量用于处理类及其实例对象的相关的数据。

  • 方法重写：如果从父类继承的方法不能满足子类的需求，可以对其进行改写，这个过程叫方法的覆盖（override），也称为方法的重写。

  • 局部变量：定义在方法中的变量，只作用于当前实例的类。

    __private_method：两个下划线开头，声明该方法为私有方法，只能在类的内部调用 ，不能在类的外部调用。

  • 实例变量：在类的声明中，属性是用变量来表示的，这种变量就称为实例变量，实例变量就是一个用 self 修饰的变量。

  • 继承：即一个派生类（derived class）继承基类（base class）的字段和方法。继承也允许把一个派生类的对象作为一个基类对象对待。例如，有这样一个设计：一个 Dog 类型的对象派生自 Animal 类，这是模拟 “是一个（is-a）” 关系（例图，Dog 是一个 Animal）。

  • 实例化：创建一个类的实例，类的具体对象。

  • 对象：通过类定义的数据结构实例。对象包括两个数据成员（类变量和实例变量）和方法。

函数/方法 作用 示例 返回值/说明
datetime.now() 获取当前本地时间 datetime.now() 返回当前本地时间，类型为 datetime.datetime
datetime.today() 获取当前日期和时间 datetime.today() 类似 now()，不带时区信息
datetime.utcnow() 获取当前 UTC 时间 datetime.utcnow() 返回 UTC 时间（无时区信息）
datetime.strptime(s, fmt) 字符串转为 datetime 对象 datetime.strptime("2024-05-12", "%Y-%m-%d") 将字符串按指定格式解析为 datetime 对象
datetime.strftime(fmt) datetime 对象转为字符串 dt.strftime("%Y-%m-%d %H:%M:%S") 按指定格式返回字符串
datetime.date() 获取日期部分 dt.date() 返回 date 对象，如 2024-05-12
datetime.time() 获取时间部分 dt.time() 返回 time 对象，如 14:30:00
datetime.replace() 替换 datetime 中的指定字段 dt.replace(hour=0, minute=0) 返回一个新的 datetime 对象
timedelta(days=1) 表示时间差 datetime.now() + timedelta(days=1) 表示一天之后的时间
datetime.timestamp() 转为时间戳 dt.timestamp() 返回 float 类型的秒数，Unix 时间戳
datetime.fromtimestamp(ts) 时间戳转为 datetime datetime.fromtimestamp(1715529600) 返回对应的本地时间
datetime.isoweekday() 获取 ISO 星期几（1-7） dt.isoweekday() 返回整数，星期一为 1，星期日为 7
datetime.weekday() 获取星期几（0-6） dt.weekday() 返回整数，星期一为 0，星期日为 6
date.today() 获取当前日期 date.today() 返回当前日期对象，类型为 datetime.date
date.fromisoformat(s) 从 ISO 格式字符串创建日期对象 date.fromisoformat("2024-05-12") 返回 date 对象
time(hour, minute, second) 创建一个时间对象 time(14, 30, 0) 返回 time 对象

0x0705 re 模块

函数/方法 作用 示例 返回值/说明
re.match(pattern, string) 从字符串起始位置匹配 re.match(r'\d+', '123abc') 匹配成功返回 Match 对象，否则返回 None
re.search(pattern, string) 在整个字符串中搜索第一个匹配 re.search(r'\d+', 'abc123def') 返回第一个匹配的 Match 对象，找不到返回 None
re.findall(pattern, string) 查找所有匹配项，返回列表 re.findall(r'\d+', 'a1b22c333') 返回所有匹配项组成的列表，如 ['1', '22', '333']
re.finditer(pattern, string) 查找所有匹配项，返回迭代器 re.finditer(r'\d+', 'a1b22') 返回一个迭代器，元素为 Match 对象
re.sub(pattern, repl, string) 替换匹配内容 re.sub(r'\d+', '#', 'a1b2c3') 返回替换后的字符串，如 'a#b#c#'
re.subn(pattern, repl, string) 替换并统计替换次数 re.subn(r'\d+', '#', 'a1b2c3') 返回 (新字符串, 替换次数)，如 ('a#b#c#', 3)
re.split(pattern, string) 按匹配内容分割字符串 re.split(r'\d+', 'a1b22c333') 返回列表，如 ['a', 'b', 'c', '']
re.compile(pattern) 编译正则表达式，生成 Pattern 对象 pattern = re.compile(r'\d+') 返回 Pattern 对象，可复用匹配方法
Match.group() 获取匹配的字符串 m = re.match(r'\d+', '123abc'); m.group() 返回匹配的字符串，如 '123'
Match.groups() 获取所有子组 m = re.match(r'(\d+)-(\w+)', '123-abc'); m.groups() 返回元组，如 ('123', 'abc')
Match.start() / Match.end() 获取匹配的起止位置 m.start() / m.end() 返回匹配字符串的起始和结束索引

0x0706A json 模块

函数/方法 作用 示例 返回值/说明
json.loads(s) 将 JSON 字符串转为 Python 对象 json.loads('{"name": "Tom", "age": 25}') 返回字典：{'name': 'Tom', 'age': 25}
json.dumps(obj) 将 Python 对象转为 JSON 字符串 json.dumps({'name': 'Tom', 'age': 25}) 返回字符串：'{"name": "Tom", "age": 25}'
json.load(fp) 从文件读取 JSON 并转为对象 json.load(open('data.json', 'r')) 返回 Python 对象（通常是字典或列表）
json.dump(obj, fp) 将对象转为 JSON 并写入文件 json.dump(data, open('data.json', 'w')) 无返回值，将数据写入文件
json.dumps(obj, indent=4) 美化格式输出 JSON 字符串 json.dumps(data, indent=4) 返回带缩进格式的字符串，便于阅读
json.dumps(obj, ensure_ascii=False) 保留非 ASCII 字符 json.dumps({'城市': '北京'}, ensure_ascii=False) 返回包含中文字符的字符串：'{"城市": "北京"}'
json.dumps(obj, sort_keys=True) 对字典键排序 json.dumps({'b': 1, 'a': 2}, sort_keys=True) 返回：'{"a": 2, "b": 1}'
json.JSONDecodeError 解码失败异常类型 try: json.loads('bad') except json.JSONDecodeError: ... 当 JSON 格式不合法时抛出该异常

0x0706B urllib 模块

函数/方法 作用 示例 返回值/说明
urllib.request.urlopen(url) 打开并读取 URL urllib.request.urlopen('https://www.example.com') 返回一个 HTTPResponse 对象，可通过 .read() 获取内容
urllib.request.Request(url, data, headers) 自定义请求（方法、头部等） urllib.request.Request(url, data, headers) 创建请求对象，可传给 urlopen() 使用
urllib.request.urlretrieve(url, filename) 下载文件 urllib.request.urlretrieve(url, 'file.html') 下载指定 URL 内容到本地文件
urllib.parse.urlparse(url) 解析 URL 成各部分 urllib.parse.urlparse('https://a.com/path?x=1#frag') 返回 ParseResult 元组，含 scheme、netloc、path、params 等信息
urllib.parse.urlencode(dict) 将字典编码为查询字符串 urllib.parse.urlencode({'a': 1, 'b': 2}) 返回字符串：'a=1&b=2'
urllib.parse.quote(string) URL 编码 urllib.parse.quote('中文') 返回 %E4%B8%AD%E6%96%87
urllib.parse.unquote(string) URL 解码 urllib.parse.unquote('%E4%B8%AD%E6%96%87') 返回 中文
urllib.parse.urljoin(base, url) 拼接完整 URL urllib.parse.urljoin('https://a.com/dir/', 'page.html') 返回完整 URL：'https://a.com/dir/page.html'
urllib.parse.parse_qs(query) 解析查询字符串为字典（值为列表） urllib.parse.parse_qs('a=1&b=2&b=3') 返回：{'a': ['1'], 'b': ['2', '3']}
urllib.parse.parse_qsl(query) 解析查询字符串为键值对列表 urllib.parse.parse_qsl('a=1&b=2') 返回：[('a', '1'), ('b', '2')]
