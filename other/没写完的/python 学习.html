Python3 和 Python2 有较大差别，该文档不适用于 Python2。

0x01 什么是 Python

Python 是一种解释型、面向对象、动态数据类型的高级程序设计语言。Python3 和 Python2 具有较大的差异，现在主要使用的是 Python3，比较稳定的版本是 3.9 版本（2025-05-07 更新），算是兼容比较多的版本。

0x02 环境搭建

我们可以使用 PyCharm 作为我们的编译器，并使用 Anaconda 作为 Python 包管理器。

由于 Python 有非常多的包（Packages，PyPI），我们需要根据不同的项目安装不同的包。但是项目 A 和项目 B 之间可能会存在包冲突，及 Package 1 和 Package 2 本身不能兼容或两者版本不能兼容，或者 Package 3 与当前 Python 版本不兼容，所以我们最好使用 Anaconda 来为每个项目管理 Python 版本和相关的包。

而 Anaconda 为每个项目隔离出来的 Python 版本和相关的包，我们称它们为一个 环境 ，所以我们应该为每一个项目先创建一个环境，再在该环境配置相关的 Python 版本和包。

具体来说（以下只是一种方式，按照自己的习惯来就好），

  • 安装 Pytharm、Anaconda，并配置好相关环境变量；

  • 对于某一个项目，为它创建对应 环境 ，并指定 Python 版本；

  • 在 Pycharm 打开已有项目或空项目，在 设置 -> 项目 -> Python 解释器 -> 添加解释器 中添加上一步创建的环境的 python.exe ，具体位置你可以在  anaconda 根目录 -> env -> your_environment_name -> bin or 当前目录  中找到。

  • 通过 pip 安装对应的包，如果涉及 Pytorch 建议使用 官方命令进行安装。如果是已有项目，检查是否存在 requirement.txt 文件。

0x03 代码规范 更优雅的代码

你也可以不遵守用 abcd 来命名，但是本来可以很优雅的。

Python 的命名 PEP 8（Python Enhancement Proposal 8） 规范，以下是核心规则和常见场景的命名约定。

0x0301 代码布局

  • 行长度：每行不超过 79 字符（代码）或 72 字符（文档字符串/注释），换行使用反斜杠 \ 或括号自然延续；

  • 空行：函数/类之间：2 行；方法/逻辑块之间：1 行；

0x0302 命名规范

  1. 命名风格：

    • 小写字母 + 下划线（snake_case）：变量、函数、方法、模块、包

    • 首字母大写（PascalCase/CamelCase）：类名、异常

    • 全大写 + 下划线：常量

  2. 避免的命名：

    • 单字符（除非是临时变量或数学符号）：i, x（可接受在循环中）
    • 保留 关键字 冲突：如 list，str（可加下划线后缀 list_），
    • 模糊缩写：tmp（可接受），但避免 fn（应为 function）

  3. 具体场景

    类型 规范 示例
    变量/函数 snake_case user_count
    类 PascalCase DatabaseEngine
    常量 UPPER_CASE MAX_RETRIES = 3
    私有成员 _single_underline _internal_data
    魔法方法 __double_underline__ `init

  4. 其他建议

    • 一致性优先：如果项目已有规范，优先遵循项目约定。

    • 描述性命名：

      • 推荐：file_path（明确）
      • 避免：fp（模糊）

    • 避免误导：

      • 布尔变量用 is_ 或 has_ 前缀：is_valid
      • 集合类型用复数：users（而非 user_list，除非强调类型）

    • 避免可变默认参数：

    编译器可以一键规范格式，但是无法一键规范命名。

0x04 基本数据类型

Python 是一种动态类型语言，声明变量不用指明类型，Python 会根据赋给变量的值自动进行推断。

Python3 中常见的六个标准数据类型：

  • 不可变类型（Immutable）： Number（数字）、String（字符串）、Tuple（元组）；

  • 可变类型（Mutable）： List（列表）、Dictionary（字典）、Set（集合）。

    不可变类型（Immutable）：创建后，值本身（内存中的内容）无法改变，「修改」操作实际是创建新对象，可作为字典的键或集合的元素；

    可变类型（Mutable）：无需创建新对象即可增删改内容，不能直接作为字典的键或集合的元素（需转为不可变类型，如 tuple 或 frozenset）。

    其中，Number（数字）包括 int、float、bool、complex（复数）四种。

0x0401 基本数据类型概览表

哈希性

  • 可哈希（Hashable）对象才能作为字典的键或集合的元素（需实现 __hash__ 方法）；
  • 是否可哈希和可变性并不相关，尽管实际上 range 对象的内容在创建后是不可变的，但 Python 仍将其归类为不可哈希类型；

常见误区

  • 元组不可变，但若包含可变元素（如列表），其内容仍可修改；
  • 变量名是引用，重新赋值（如 x = 10 → x = 20）是改变引用，而非修改对象；

collections.namedtuple 带字段名的元组 不可变

0x0402 基本数据类型

0x0402A Number bool

Python3 支持 int、float、bool、complex（复数），绝大部分都和其他编程语言一样，在此不做过多赘述，部分值得注意的是：

  • 数值的除法包含两个运算符：/ 返回一个浮点数，// 返回一个整数。// 得到的并不一定是整数类型的数，它与分母分子的数据类型有关系。

  • 在 Python 中，number（数字类型，如 int、float）是不可变（immutable）的。这意味着一旦创建了一个数字对象，它的值就不能被修改。

    a = 10 a = 20 # 这里不是修改 10，而是让 a 指向一个新的数字对象 20

  • 所有非零的数字和非空的字符串、列表、元组等数据类型都被视为 True，只有 0、空字符串、空列表、空元组等被视为 False。因此，在进行布尔类型转换时，需要注意数据类型的真假性。

0x0402C List []

list 是 Python 的“万能动态数组”，灵活但需注意 复制、遍历修改和性能 问题！和字符串一样，列表同样可以被索引和截取，列表被截取后返回一个包含所需元素的新列表。列表截取的语法格式变量[头下标: 尾下标: 步长]，索引值以 0 为开始值，-1 为从末尾的开始位置。

  • List 可以使用 + 操作符进行拼接；

  • List 中的元素是可以改变的，是不可哈希的；

  • List 相比字符串内置了有很多方法，例如 append()、pop() 等等；

    操作 示例 说明
    添加元素 lst.append(x) 末尾添加
    插入元素 lst.insert(i, x) 在索引 i 处插入
    删除元素 lst.remove(x) 删除第一个匹配的 x
    合并列表 lst1 + lst2 返回新列表
    列表生成式 [x*2 for x in lst if x>0] 快速生成新列表

  • append() 是 O(1) 操作（高效）；insert(0, x) 是 O(n) 操作（头部插入慢，需移动所有元素）；需要频繁头部操作时，考虑用 collections.deque。

非常值得注意的是：

  1. 列表复制问题：

    • 直接赋值 = 是 浅拷贝（共用同一内存），修改会影响原列表！

    • 正确做法：用 copy() 或 list() 复制新列表。

  2.  循环中修改列表：

    • 边遍历边修改列表可能导致意外结果或报错。

    • 正确做法：遍历副本或新建列表。

0x0402D tuple ()

tuple 是 Python 中的一种不可变序列，可以理解为一个 “一旦创建就不能修改的固定清单”。

  • 不可变（Immutable）：创建后 不能增、删、改 元素。
  • 有序：元素按添加顺序排列，可以通过索引（变量[头下标: 尾下标: 步长]）访问。
  • 可存任意类型：和列表一样，支持数字、字符串、列表等。
  • 允许重复：同一个值可以出现多次。

非常值得注意的是：

  1.  单元素元组的陷阱：如果元组只有一个元素，必须加逗号，否则会被当成普通变量！

  2. 元组内的可变对象：元组本身不可变，但如果元素是可变对象（如列表），内部内容仍可修改！

  3. 性能比列表略高：

    • 由于不可变性，元组的 存储和访问速度比列表快，适合存储常量数据。

    • 在需要哈希（如字典的键）时，元组比列表更合适（列表不可哈希）。

0x0402E set

set 是 Python 中的一种无序、不重复的数据容器，可以理解为一个 “自动去重的数学集合”。

  • 无序：元素没有固定顺序（不能通过索引访问）。
  • 唯一性：自动去除重复值（例如 {1, 2, 2} 会变成 {1, 2}）。
  • 可变（Mutable）：可以增删元素，但元素本身必须不可变（如数字、字符串、元组）。
  • 高效查询：用哈希表实现，检查元素是否存在比列表快得多（O(1) 时间复杂度）。
操作 示例 说明
添加元素 s.add(x) 添加单个元素
删除元素 s.remove(x) 删除指定元素（不存在则报错）
集合运算 s1，s2 返回新集合
s1 & s2（交集）
s1 - s2（差集）
快速去重 list(set([1, 2, 2])) 列表 → 集合 → 列表

值得注意的是：

  1. 空集合必须用 set()，不能用 `{}

  2. 集合只能存不可变对象

    • 合法：数字、字符串、元组（如果元组内不含可变对象）。

    • 非法：列表、字典、其他集合（因为它们可变）。

  3. 集合是无序的，不要依赖顺序，每次打印的顺序可能不同（Python 3.7+ 后字典有序，但集合仍无序）。

  4. 性能陷阱

    • in 操作极快（O(1)），适合频繁查询。

    • 遍历比列表慢（因哈希表结构），如需有序遍历，建议转列表：

0x0402F Dictionary

dict 是 Python 中的 键值对集合，可以理解为一个 “现实生活中的字典”（通过“单词”查“释义”），dict 是 Python 的“万能查找表”，用键快速找值，适合存结构化数据或高频查询！记住键必须不可变，访问不存在的键用 .get() 防报错。

  • 键值对结构：每个元素由 key: value 组成（例如 {"name": "Alice", "age": 25}）。
  • 键必须不可变：key 可以是 str、int、tuple 等不可变类型，但不能是 list、dict 等可变类型。
  • 无序（Python 3.7+ 后按插入顺序保存，但不要依赖顺序）。
  • 可变：可以随时增删改键值对。
  • 查询极快：通过 key 找 value 的时间复杂度是 O(1)（哈希表实现）。
操作 示例 说明
获取所有键 d.keys() 返回键的视图（可转列表）
获取所有值 d.values() 返回值的视图
获取所有键值对 d.items() 返回 (key, value) 对
合并字典 d1.update(d2) 将 d2 合并到 d1
字典生成式 {k: v*2 for k, v in d.items()} 快速生成新字典
获取值 d.get("key", "default_value")

值得注意的是：

  1. 访问不存在的键会报错，应该用 .get() 方法提供默认值。 ：

  2. 字典的键必须不可变：

    • 合法键：str、int、float、tuple（如果元组内不含可变元素）。

    • 非法键：list、dict、set。

  3. 字典是无序的（即使 Python 3.7+ 保留插入顺序），

    • 如果需要有序字典，用 collections.OrderedDict。

    • 不要依赖遍历顺序（除非明确使用排序）。

  4. 避免在循环中修改字典，边遍历边增删字典可能导致意外结果，应该遍历副本或先收集要修改的键；

0x0402G bytes

Python 中的 bytes（字节）简单解释，bytes 是 Python 中表示 二进制数据 的不可变序列，可以理解为一串 “0 和 1 的原始数据”（比如文件、网络传输的原始格式）。

  • 不可变（Immutable）：创建后不能修改（类似 tuple）。
  • 存储二进制数据：如图片、音频、加密数据等。
  • 与 str 的区别：
    • str 是 文本（如 "hello"），用 Unicode 编码。
    • bytes 是 二进制（如 b"hello"），用 ASCII/UTF-8 等编码的原始字节。

bytes vs bytearray

特性 bytes bytearray
可变性 ❌ 不可变 ✅ 可变
语法 b"ABC" 或 bytes() bytearray(b"ABC")
用途 存储固定二进制数据 需要修改的二进制数据

什么时候用 bytes？

  1. 处理文件/网络数据：如图片、音频、PDF 等二进制文件。
  2. 加密/哈希运算：如 hashlib.md5(b"data") 需要输入 bytes。
  3. 底层协议通信：如 TCP/UDP 传输原始数据。
